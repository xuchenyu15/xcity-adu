# 第一阶段完成情况说明

本说明用于对照里程碑目标，记录当前仓库在“西雅图试点范围内”的第一阶段完成情况与可交付物。

## 中期里程碑（完成情况）

- 西雅图市试点范围内核心判断链路已跑通  
  - 后端已提供统一入口 `/api/lookup`，可在西雅图地区适配器下完成：地址解析 → 地块/周边要素抓取 → 规则计算 → 结构化结果返回。
  - 主要代码位置：`server-dotnet/XBuildApi/Controllers/LookupController.cs`、`server-dotnet/XBuildApi/Site/RegionAdapters.cs`、`server-dotnet/XBuildApi/Lookup/PlanBuilder.cs`。

- 可稳定输出：地块边界、可建区域、通道判断、预制房放置逻辑及结果分类  
  - 地块边界：`subjectParcel`（GeoJSON Feature）  
  - 可建区域：`computed.buildableArea`（GeoJSON，支持多边形/洞）与 `computed.buildablePolygonFt`（英尺坐标多边形）  
  - 通道判断：`computed.drivewayCorridorFt`（英尺坐标多边形/走廊）  
  - 放置逻辑/结果：`computed.aduFits`（不同模块尺寸的可放置性结果）、`computed.aduPlacementArea`（中心点可落点区域，若不可放置则置空）  
  - 主要代码位置：`server-dotnet/XBuildApi/Controllers/LookupController.cs`（ComputeComputed 聚合）、`server-dotnet/XBuildApi/Lookup/PlanBuilder.cs`（plan 构建与几何计算）。

- 已形成可用于前端接入的基础结构化输出  
  - 返回结构已稳定为 `SiteLookupResponse`：`SubjectParcel / SubjectBuildings / Nearby* / Computed / AiParcelInfo`。  
  - 结构定义位置：`server-dotnet/XBuildApi/Site/SiteModels.cs`。

- 已形成第一版 2D 图输出框架  
  - 前端 2D（SVG Map）已能渲染：地块边界、道路/周边要素、既有建筑、可建区（含洞）、退尺/切割区域等叠加层。  
  - 主要代码位置：`figma/src/components/studio/SiteFeasibility.tsx`、`figma/src/components/studio/SiteVisualizer.tsx`（2D 模式）。

## 第一阶段结束（完成情况）

- 抽样案例验证  
  - 仓库内已整理一批测试地址（见根目录 `测试地址.txt`），用于覆盖“有车库/无车库/放得下/放不下”等典型场景的抽样验证。

- 2D 输出  
  - `SiteFeasibility` 与 `SiteVisualizer` 的 2D 视图已具备可用输出：地块/可建区/障碍/退尺及标注叠加显示。
