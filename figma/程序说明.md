# 程序说明（Studio / Site 相关）

本项目的 Studio（Design Studio / Site Feasibility / Site Visualizer）围绕“地址 → lookup → 地块/建筑 → ADU 摆放 → 3D 展示/测量”这一条链路工作。  
其中 localStorage 主要用于两类数据：

1. StartPage 的 lookup 结果缓存（减少重复请求、支持“最近地址”一键复用）
2. SiteFeasibility / SiteVisualizer 的 ADU 摆放持久化（位置/角度/外挂/露台/可建区）

---

## 2. 数据流（从地址到 Studio）

### 2.1 StartPage（地址与 lookup）

1. 用户输入地址或点击最近地址
2. 生成/解析 `lookupId`（稳定地块 id）
3. 优先从 localStorage 读 `xhomes.lookup:<lookupId>`，命中则直接进入 Studio
4. 未命中再请求接口拿 lookup，并写入：
   - `xhomes.lookup`（最近一次 lookup）
   - `xhomes.lookup:<lookupId>`（按地块 id 缓存）
   - `xhomes.recentAddresses`（最近地址列表）

### 2.2 Studio（地块/建筑与 ADU）

Studio 页面会消费 `lookup` 数据并渲染：
- 地块边界：`lookup.subjectParcel`
- 本地建筑：`lookup.subjectBuildings`
- 邻居地块/建筑：`lookup.nearbyParcels` / `lookup.nearbyBuildings`（不同 provider 可能字段不同）

ADU 的放置/旋转/外挂/露台会写入 localStorage，以便刷新后恢复。

## 3. localStorage 约定

### 3.1 语言相关

- `i18nextLng`
  - i18next 自动写入的当前语言（库自身使用）
- `xhomes.lang`
  - 项目自定义语言偏好（用于 request.lang 等）

### 3.2 最近用过地址（StartPage）

- `xhomes.recentAddresses`
  - 结构：`[{ address: string, lookupId?: string, updatedAt?: number }]`
  - `address`：展示给用户的文本
  - `lookupId`：稳定地块 id（取 `subjectParcel.properties.fields.ll_uuid`）
    - 规范化：trim / lowercase / 压缩空白
    - 例：`kc:2113200065`
  - 逻辑：点击“最近用过”会优先用 `lookupId` 直接读 `xhomes.lookup:<lookupId>`，不必再次请求后端

### 3.3 lookup 结果（StartPage → Studio）

- `xhomes.lookup`
  - “最近一次 lookup”的完整数据（用于 Studio 启动/回填）
  - 结构类似：`{ request, region, subjectParcel, subjectBuildings, nearbyParcels, ... }`

- `xhomes.lookup:<lookupId>`
  - 按地块 id 缓存的 lookup 结果
  - 当前写入格式：`{ data, updatedAt, address }`
  - 例：`xhomes.lookup:kc:2113200065`
  - 安全校验：读取缓存时会用 `data` 里派生出来的 `ll_uuid` 再对比 `lookupId`，不一致则视为无效（避免错配）

### 3.4 ADU 摆放/外挂/露台持久化（SiteFeasibility / SiteVisualizer）

- `xhomes.aduPlacement:<lookupId>`
  - 按地块 id 保存 ADU 的位置/角度与 addons
  - 例：`xhomes.aduPlacement:kc:2113200065`
  - payload 常见字段：
    - `cxPx/cyPx`：当前画布坐标（兜底）
    - `cxRel/cyRel`：相对 lot bounds 的归一化坐标（0~1，推荐主恢复字段）
    - `cxM/cyM`：Web Mercator 投影坐标（单位：米，兜底恢复字段）
    - `cxFt/cyFt`：相对 LOT 坐标的英尺坐标（无 bboxMercator 时兜底）
    - `rotationDeg`：ADU 旋转角
    - `endAddon`：`none | end-right | end-left | end-both`
    - `hasBalcony`：是否有入户露台
    - `selectedBuildablePolyIdx`：当前选中的可建区
    - `updatedAt`：写入时间戳

### 3.5 注意

- `xhomes.aduPlacement:default`
  - 旧逻辑在 parcel id 未 ready 时写入的临时 key，容易造成“临时主体/推荐位/随机性”
  - 当前代码已禁止写入，并会在 key ready 后清理

## 4. 3D 展示与交互（SVG 等距伪 3D）

### 4.1 3D 技术路线

- `SiteFeasibility.tsx` 与 `SiteVisualizer.tsx` 的“3D Massing”主体都是 **SVG 等距伪 3D**
  - 等距投影：`x=(x-y)*cos30`, `y=(x+y)*sin30 - z`
  - 拖拽：`dx → yaw`，`dy → pitch`（带 damp + clamp）

### 4.2 同屏测量

- 测量不跳转页面，在同一画面点两点测距离
- 3D 测量支持三维直线距离（需要正确的 transform 反解 + scale 换算）

## 5. GLB 模型叠加（保持 SVG 3D 风格不变）

### 5.1 目标

- 保持 SVG 3D（地块/周边建筑）不变
- 用透明 `canvas` 叠加渲染 `module.glb`（ADU 位置），用于替代“蓝色立方体 ADU/外挂/露台”

### 5.2 模型静态资源放置

要让 GLB 被打包进 `static-assets`，需要放到：

- `figma/public/static-assets/models/<ModelName>/module.glb`
- `figma/public/static-assets/models/<ModelName>/tileset.json`（如需）

例如：

- `figma/public/static-assets/models/Model2/module.glb`
- `figma/public/static-assets/models/Model2/tileset.json`

构建后路径：

- `figma/build/static-assets/models/Model2/module.glb`

### 5.3 语义对象约定（用于对齐/显隐）

GLB 内部对象命名会影响对齐与显隐逻辑（按约定匹配）：

- `墙体`：作为 ADU 主体对齐基准（尺寸拟合、落地、中心对齐）
- `地板`：用于决定入户侧/主体朝向（即使没有露台也用于朝向计算）
- `建筑3`：右侧外挂（随 `endAddon` 显示/隐藏）
- `建筑4`：左侧外挂（随 `endAddon` 显示/隐藏）
- `建筑1` / `建筑2`：永远隐藏
- `屋顶*`：可在 3D 画面切换显示（若模型中存在）

### 5.4 屋顶显示开关

- 当模型里存在 `屋顶`（或以 `屋顶` 开头）对象时，3D 控制条会出现 `Roof` 开关
- 开关仅影响 GLB 内 `屋顶*` 对象的 `visible`

## 7. 常见问题排查

- “3D 看不见/空白”
  - 多半是初次挂载时容器尺寸为 0，导致渲染尺寸/相机初始化异常（需要等到有有效宽高再 setSize）
- “Map is not a constructor”
  - 常见于把 `lucide-react` 的 `Map` 图标 import 覆盖了全局 `Map`（建议命名为 `MapIcon`）
- “后面的房子压住模型”
  - 需要用“地面层深度”做前后分层（不要把楼高加进分层判断）
